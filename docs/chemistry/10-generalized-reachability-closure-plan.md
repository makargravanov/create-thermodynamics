# План закрытия дыр достижимости органики

Цель этого документа: довести органическую модель до состояния, где статические и динамические органические вещества становятся достижимы из простых предшественников через общие химические слои, а не через частные рецепты для отдельных веществ.

Этот план продолжает:

- `08-generalized-organic-synthesis-plan.md`;
- `09-complete-organic-synthesis-implementation-plan.md`.

Главная задача: использовать список недостижимых веществ как диагностический отчет о недостающих классах реакций. Нельзя превращать этот список в набор частных генераторов.

## Исходная картина

После исправления ошибки с TMS-де-защитой проверка достижимости из простого набора веществ дала:

- стартовых простых веществ: 32;
- статических органических целей Destroy: 83;
- достижимо на глубине 3: 53;
- недостижимо на глубине 3: 30;
- сгенерировано: 27 542 вещества и 31 272 реакции;
- до неподвижной точки проверка не дошла, очередь еще содержала 27 194 задачи.

Важно: это не доказывает, что вещество невозможно синтезировать вообще. Это показывает, какие классы превращений отсутствуют или пока не дают короткий путь из простых предшественников.

## Главный запрет

Нельзя закрывать эти цели частными реакциями.

Нельзя:

```rust
generate_methyl_acetate(...)
generate_acetone_cyanohydrin(...)
generate_isopropanol(...)
generate_trimethyl_borate(...)
if substance.id == "destroy:aspirin" { ... }
```

Нужно:

```rust
generate_carbonyl_cyanohydrin_formation(...)
generate_acyl_transfer_to_oxygen(...)
generate_hydration_of_alkenes(...)
generate_boron_esterification(...)
generate_nucleophile_c1_alkylation(...)
```

Если в результате такого общего преобразования получается конкретное статическое вещество, это нормальный результат. Но код не должен знать, что оно было целью.

## Архитектурные правила

1. Источник истины для реакционной способности - типизированный реакционный центр, а не имя вещества.
2. Генератор описывает класс преобразований, а не маршрут к одной цели.
3. Продукт строится из фактических атомов реагентов через `MolecularEditor`.
4. Побочные продукты создаются только из реально удаленных или перенесенных атомов.
5. Водороды учитываются только как явные атомы.
6. Нельзя отключать проверку массы, заряда, валентности, связности, ароматичности и стереохимии.
7. Структурно возможная реакция должна генерироваться независимо от обычных условий среды; условия влияют на скорость и распределение каналов.
8. Если генератор нашел подходящий центр, но преобразование ломает граф, это ошибка, а не пустой список реакций.
9. Если структура химически не подходит, допустимо вернуть отсутствие реакции с диагностически понятной причиной.
10. Каждый новый генератор должен иметь минимум два положительных теста на разные субстраты одного класса и минимум один отрицательный тест на похожий неподходящий фрагмент.
11. Проверочный тест не должен ограничиваться одной целевой молекулой из списка недостижимых.
12. Планировщик должен использовать эти слои для объяснения путей и обрывов, но не должен мутировать основной реестр игрока.

## Этап 0. Зафиксировать измерение достижимости

Перед добавлением новых слоев нужно сделать проверку воспроизводимой.

Сделать:

1. Добавить тест или вспомогательную проверку достижимости статических органических целей из стандартного набора простых веществ.
2. Отчет должен показывать:
   - количество стартовых веществ;
   - количество целей;
   - достижимые цели;
   - недостижимые цели;
   - для каждой недостижимой цели: есть ли входящая реакция;
   - если входящая реакция есть, какие предшественники недостижимы;
   - сколько веществ и реакций добавлено;
   - дошла ли генерация до неподвижной точки.
3. Отчет не должен расширять основной реестр игры.
4. Отчет должен работать через отдельное рабочее состояние планировщика или проверки.

Критерий готовности:

- текущие 30 недостижимых целей воспроизводятся как диагностический отчет;
- после каждого нового слоя видно, какие цели стали достижимы и почему;
- проверка не требует полного расширения всего химического пространства.

## Этап 1. C1-алкилирование и метилирующие реагенты

Закрываем класс проблем:

- `iodomethane`;
- `methylamine`;
- `trimethylamine`;
- `trimethylphosphine`;
- часть путей к N-метилированным гетероциклам.

Нужные центры:

- `C1ElectrophileCenter`;
- `AlkylHalideCenter`;
- `SulfonateEsterLeavingGroupCenter`, если сульфонаты есть в модели;
- `NucleophilicNitrogenCenter`;
- `NucleophilicPhosphorusCenter`;
- `AlcoholOxygenCenter`;
- `ThiolateSulfurCenter`.

Генераторы:

- образование метилгалогенидов из метанола и галогенводородов или других общих галогенирующих условий;
- алкилирование аммиака, аминов, фосфинов, спиртов и тиолов C1-электрофилами;
- ступенчатое алкилирование, где моно-, ди- и триалкилированные продукты возникают из одного механизма;
- конкуренция переалкилирования через селективность и доступность нуклеофильного центра.

Правила:

- не создавать `methylamine` как отдельную цель;
- не проверять имя `trimethylamine`;
- каждый следующий алкилированный продукт должен снова распознаваться как нуклеофильный центр, если он химически способен реагировать;
- четвертичные соли должны получаться только если заряд, валентность и противоион описаны корректно.

Тесты:

- метанол + галогенводород дает метилгалогенид как частный случай общего замещения спиртов;
- аммиак + метилгалогенид дает метиламин;
- метиламин может дальше давать диметиламин и триметиламин тем же кодом;
- фосфин или замещенный фосфин алкилируется тем же механизмом;
- неподходящий C1-фрагмент без уходящей группы не распознается как электрофил.

## Этап 2. Циангидрины и нитрильные предшественники

Закрываем класс проблем:

- `acetone_cyanohydrin`;
- `aibn`;
- часть путей к `adiponitrile`.

Нужные центры:

- `CarbonylCenter`;
- `CyanideNucleophileCenter`;
- `CyanohydrinCenter`;
- `AzoInitiatorPrecursorCenter`, если модель поддерживает дальнейшее превращение циангидринов в азосоединения.

Генераторы:

- присоединение цианида к альдегидам и кетонам;
- протонирование алкоксида после присоединения;
- обратимое равновесие циангидрина через селективность и условия;
- дальнейшие общие превращения циангидринов, если они уже нужны для азосоединений.

Правила:

- циангидрин строится из реального карбонильного углерода, карбонильного кислорода, цианида и протона;
- нельзя вставлять готовый `C(OH)(CN)` фрагмент без источника атомов;
- стереохимия нового центра должна быть задана как распределение изомеров, если атака не селективна.

Тесты:

- формальдегид, ацетальдегид и ацетон дают соответствующие циангидрины одним генератором;
- карбоновая кислота или эфир не распознается как обычный карбонил для этой реакции;
- при отсутствии цианида реакция не имеет доступного реагента;
- `acetone_cyanohydrin` становится достижимым без частного генератора.

## Этап 3. Ацильный перенос, ангидриды, ацетилирование и салицилаты

Закрываем класс проблем:

- `acetic_anhydride`;
- `methyl_acetate`;
- `methyl_salicylate`;
- `aspirin`;
- `di_tert_butyl_dicarbonate`;
- `benzyl_chloroformate`.

Нужные центры:

- `CarboxylicAcidCenter`;
- `CarboxylateCenter`;
- `AcylChlorideCenter`;
- `AcidAnhydrideCenter`;
- `ActivatedCarbonateCenter`;
- `AlcoholOxygenCenter`;
- `PhenolOxygenCenter`;
- `CarboxylateOxygenCenter`;
- `AcylTransferAcceptorCenter`.

Генераторы:

- образование ацилхлоридов и других активированных ацильных доноров из кислот;
- образование ангидридов из кислоты и активированного ацильного производного;
- ацильный перенос на спирты, фенолы, амины и карбоксилаты;
- образование карбонатов и хлорформиатов через общие карбонильные переносчики;
- снятие и установка Boc/Cbz-подобных защитных групп как обычные графовые преобразования.

Правила:

- `aspirin` должен получаться как ацильный перенос на фенольный кислород салицилатного фрагмента, а не как особый продукт;
- `methyl_acetate` и `methyl_salicylate` должны быть частными случаями этерификации или ацильного переноса;
- `di_tert_butyl_dicarbonate` должен возникать из общего слоя карбонатов, если есть доступные трет-бутильные фрагменты;
- защитная группа не является флагом, она должна реально менять граф и закрывать центр.

Тесты:

- уксусная кислота и метанол дают метилацетат;
- салицилатный фенол и ацильный донор дают ацетилированный фенол;
- обычный спирт и фенол используют один слой акцепторов, но имеют разные селективные веса;
- кислота без активации реагирует медленнее или требует соответствующих условий, а не исчезает из пространства;
- продукты проходят проверку массы и заряда.

## Этап 4. Разветвленные спирты и гидратация алкенов

Закрываем класс проблем:

- `isopropanol`;
- `tert_butanol`;
- часть путей к алкенам, эфирам и защитным группам.

Нужные центры:

- `AlkeneCenter`;
- `CarbocationLikeSiteDescriptor`;
- `AlcoholCenter`;
- `LeavingGroupAlcoholCenter`;
- `BranchedCarbonCenter`.

Генераторы:

- кислотная гидратация алкенов;
- гидроборирование-окисление, если соответствующий борный слой готов;
- замещение третичных галогенидов водой или спиртами через общий механизм;
- дегидратация спиртов в обратную сторону через существующую селективность.

Правила:

- не писать генератор `generate_isopropanol`;
- направление Марковникова или анти-Марковникова задается селективностью каналов;
- новый стереоцентр дает несколько веществ через распределение продуктов, а не метку “смесь”.

Тесты:

- пропен дает изопропанол в кислотной воде;
- изобутен дает трет-бутанол;
- терминальный и внутренний алкен дают разные каналы;
- без воды или протонной среды скорость должна быть мала или нулевая через селективность.

## Этап 5. Диены, сопряжение и малые углеводородные строительные блоки

Закрываем класс проблем:

- `butadiene`;
- `isoprene`;
- часть путей к полимеризации и терпеноподобным структурам.

Нужные центры:

- `AllylicAlcoholCenter`;
- `AllylicHalideCenter`;
- `ConjugatedDieneCenter`;
- `DehydrogenationSite`;
- `AlkeneCouplingSite`.

Генераторы:

- дегидратация диолов или аллильных спиртов до диенов;
- дегидрирование алкенов до сопряженных диенов при подходящих условиях;
- олигомеризация малых алкенов с последующей дегидратацией или дегидрированием, если общий слой C-C связи готов;
- изомеризация алкенов через общий аллильный механизм.

Правила:

- нельзя вставлять готовый бутадиеновый граф;
- сопряжение должно распознаваться по фактическому графу;
- E/Z-каналы должны идти через существующую стереохимию и распределение продуктов.

Тесты:

- подходящий C4-предшественник дает бутадиен;
- подходящий разветвленный C5-предшественник дает изопрен;
- несопряженный продукт и сопряженный продукт получают разные селективные веса;
- отсутствие нужных явных водородов запрещает дегидрирование.

## Этап 6. Бензильная и ароматическая боковая цепь

- `phenylacetone`;
- `benzyl_chloroformate`;
- `toluene_diisocyanate`;
- часть путей к индолам, ароматическим аминам и защитным группам.

Нужные центры:

- `BenzylicCarbonCenter`;
- `ArylMethylCenter`;
- `BenzylicHalideCenter`;
- `ArylSideChainCarbonylCenter`;
- `ArylIsocyanatePrecursorCenter`;
- `AromaticDirectedSubstitutionCenter`.

Генераторы:

- бензильное галогенирование и замещение;
- окисление/восстановление бензильной боковой цепи;
- C-C удлинение боковой цепи через общие карбонильные, енолятные или органометаллические механизмы;
- образование арил-изоцианатов из аминов или соответствующих активированных центров;
- направленное ароматическое замещение с позиционными каналами.

Правила:

- толуол, ксилолы и замещенные арены должны работать одним слоем;
- `toluene_diisocyanate` не должен быть особым случаем;
- позиционные изомеры должны быть отдельными каналами с весами, а не случайным первым найденным продуктом.

Тесты:

- арилметильный фрагмент распознается как бензильный центр;
- замещенный арилметил дает несколько позиционных или боковых продуктов, если окружение несимметрично;
- ароматическое кольцо без боковой цепи не распознается как бензильный центр;
- продукты проходят классификацию ароматичности.

- `DinitrileCenter`;

Генераторы:

- построение динитрилов из дигалогенидов и цианида;

Тесты:

- один динитрил восстанавливается в соответствующий диамин;
- другой динитрил того же класса проходит тем же механизмом;
- диамин превращается в ди-изоцианат тем же кодом, что обычный амин в изоцианат;
- планировщик показывает C6-цепь как путь из более малых строительных блоков, если все шаги доступны.

- переэтерификация борных эфиров;
- образование силикогалогенидов из силанольных или силоэфирных предшественников, если такие центры есть;
- установка и снятие силильных защитных групп только при совпадении реального силильного фрагмента;
- галогенидный обмен у кремния как общий механизм, если масса и заряд сохраняются.

Правила:

- TMS-де-защита должна применяться только к реальному trimethylsilyl-фрагменту;
- нельзя считать любой `O-Si...` TMS-эфиром;
- борный или кремниевый реагент должен строиться из реальных атомов, а не появляться как условный внешний участник без состава;
- для неизвестных борных/кремниевых окружений нужно возвращать явную диагностическую ошибку или химически обоснованное отсутствие пути.

Тесты:

- несколько спиртов образуют борные эфиры одним генератором;
- trimethylsilyl-эфир снимается с образованием правильного кремниевого побочного продукта;
- не-TMS силильный эфир не использует TMS-путь;
- продукты с бором и кремнием проходят массу, заряд и валентность.

## Этап 9. Напряженные и специальные циклические системы

Закрываем класс проблем:

- `cubane`;
- `cubanedicarboxylic_acid`;
- `cyclopentadienide`.

Этот слой нельзя делать как набор рецептов под кубан. Но надо признать, что напряженные системы требуют отдельной общей поддержки кольцевой химии.

Нужные центры:

- `CycloadditionDieneCenter`;
- `CycloadditionDienophileCenter`;
- `PhotochemicalRingContractionCenter`;
- `StrainedRingCenter`;
- `CyclopentadieneAcidBaseCenter`;
- `BridgeheadFunctionalizationCenter`.

Генераторы:

- Дильс-Альдер и обратимые циклоприсоединения как общий слой;
- фотохимические перестройки и сокращения колец как общий слой, если уже есть условия света;
- образование и депротонирование циклопентадиенильных систем;
- функционализация напряженных каркасных систем через общие центры.

Правила:

- кубан может быть проверочным результатом, но не целью генератора;
- кольцевая напряженность должна быть свойством графа или дескриптора, а не названием вещества;
- фотохимические пути должны использовать существующий `requires_uv` или более общий профиль света.

Тесты:

- один простой диен и один диенофил дают циклоприсоединение;
- замещенный диен дает замещенный циклопродукт тем же механизмом;
- циклопентадиенильный анион получается через кислотно-основный центр, а не по имени;
- невозможная напряженная структура не создается без проверки валентности и связности.

## Этап 10. Планировщик как инструмент поиска дыр

После добавления каждого слоя планировщик должен объяснять, что изменилось.

Доработать `synthesis.rs`:

- отдельное рабочее состояние планировщика;
- вход: целевое вещество или граф;
- необязательный вход: доступные вещества;
- обратный поиск от цели;
- прямое расширение только внутри рабочего состояния;
- оценка стоимости пути;
- штраф за опасные, сухие, жесткие, фотохимические, высокотемпературные и плохо селективные условия;
- учет фазового разделения и очистки;
- объяснение, каких классов предшественников не хватает;
- объяснение, где нет реакции в модели;
- ленивую выдачу `top_k` путей с возможностью продолжить поиск.

Планировщик не должен:

- мутировать `DynamicChemistryRegistry` игрока;
- добавлять вещества в основной реестр;
- обещать синтез, если соответствующей реакции нет;
- выбирать реакцию вместо симуляции смеси;
- расширять весь химический мир без входных веществ или цели.

Критерий готовности:

- для каждой недостижимой цели планировщик может сказать: “нет входящей реакции”, “есть реакция, но недостижим предшественник”, “есть путь, но условия дорогие/опасные”;
- после добавления нового слоя отчет показывает смещение обрыва глубже по цепочке;
- если путь реально появился, он выводится через общие классы реакций.

## Этап 11. Сводные проверки против частных решений

Добавить проверки, которые предотвращают деградацию архитектуры.

Проверки:

1. Поиск запрещенных имен в названиях генераторов и производственном коде:
   - `caffeine`;
   - `indole`;
   - `uracil`;
   - `xanthine`;
   - имена из текущего списка недостижимых целей, если они используются не в тесте или документации.
2. Проверка, что каждый новый генератор имеет минимум два положительных теста на разные субстраты одного класса.
3. Проверка, что каждый новый центр имеет отрицательный тест на похожий неподходящий фрагмент.
4. Проверка, что продукты новых генераторов проходят:
   - массу;
   - заряд;
   - валентность;
   - связность;
   - ароматичность;
   - стереохимию;
   - фазовую доступность, если реакция фазозависима.
5. Проверка, что генератор не использует скрытые водороды.
6. Проверка, что реакция с неподходящими условиями не исчезает из пространства, если структура возможна, а получает низкую скорость или малый вес канала.

## Этап 12. Проверочные семейства вместо проверочных веществ

Каждая закрытая дыра должна проверяться семейством.

Семейства:

- C1-алкилирование: аммиак, первичный амин, вторичный амин, фосфин, спирт;
- циангидрины: формальдегид, ацетальдегид, ацетон, замещенный кетон;
- ацильный перенос: спирт, фенол, амин, карбоксилат;
- гидратация алкенов: терминальный алкен, внутренний алкен, разветвленный алкен;
- диены: линейный C4, разветвленный C5, замещенный диен;
- бензильная химия: толуолоподобный, замещенный толуолоподобный, гетероарилметильный;
- динитрилы/диамины: C4, C5, C6 аналоги;
- бор/кремний: несколько спиртов и несколько силильных окружений;
- напряженные кольца: несколько диенов и диенофилов.

Критерий готовности:

- конкретная цель из старого списка становится достижимой как один представитель семейства;
- добавление нового похожего исходного вещества автоматически дает новые продукты;
- тесты не проверяют только одно имя вещества.

## Признаки готовности

Минимальная готовность:

- текущий список из 30 целей превращен в отчет по классам недостающих реакций;
- новые генераторы не используют имена целевых веществ;
- хотя бы C1-алкилирование, циангидрины и ацильный перенос закрыты общими слоями;
- планировщик объясняет оставшиеся обрывы.

Хорошая готовность:

- большая часть из 30 целей достижима из простого набора веществ;
- недостижимыми остаются только химически более специальные классы, например напряженные каркасы или сложные промышленные цепочки;
- для каждого нового слоя есть проверка на семейство веществ;
- условия и селективность влияют на путь, но не маскируют структурные ошибки.

Полная готовность:

- все 83 статические органические цели Destroy достижимы либо из простого набора, либо планировщик честно объясняет, какой простой неорганический/органический предшественник отсутствует;
- новые динамические аналоги становятся достижимыми без нового кода;
- планировщик может предложить несколько путей и показать, где путь дорогой, опасный или плохо селективный;
- в производственном коде нет генераторов, написанных под одну целевую молекулу;
- система сохраняет массу, заряд, валентность, связность, ароматичность, стереохимию и фазовую доступность на каждом шаге.
