# План обобщенной органики

Цель: закрыть дыры в синтезе сложной органики не частными рецептами, а обобщенными генераторами. Кофеин, индол, пурины, пиримидины и похожие вещества должны получаться как следствие реакционных центров и графовых преобразований, а не как специально зашитые молекулы.

## Главный принцип

Запрещено добавлять генератор, который узнает конкретное вещество, конкретный готовый граф или конкретный путь синтеза.

Правильная единица модели:

- реакционный центр;
- локальное окружение центра;
- допустимое графовое преобразование;
- условия, влияющие на скорость и распределение каналов;
- проверка массы, заряда, валентности, связности, ароматичности и стереохимии.

Неправильно:

```rust
generate_indole_from_phenylhydrazone(...)
generate_caffeine_from_xanthine(...)
generate_uracil_from_urea(...)
```

Правильно:

```rust
generate_hydrazone_aryl_annulation(...)
generate_amide_n_alkylation(...)
generate_bis_nucleophile_dicarbonyl_condensation(...)
```

Если в результате такого преобразования получается индол, кофеин, урацил или ксантин, это результат работы общей модели, а не отдельный путь.

## Этап 1. Типизированные центры для C/N-строителей

Добавить реакционные центры, которые описывают не конкретные вещества, а классы фрагментов.

Нужные центры:

- `HydrazoneCenter`: `C=N-N`, связь с углеродным заместителем и азотным заместителем.
- `ArylHydrazoneCenter`: гидразон, где один заместитель азота является арильным или гетероарильным фрагментом.
- `BisNucleophileCenter`: два нуклеофильных атома в одном фрагменте, обычно `N-C-N`, `N-N`, `N-C(=X)-N`.
- `DicarbonylElectrophileCenter`: 1,3- и 1,4-дикарбонильные системы.
- `ActivatedMethyleneCenter`: углерод между двумя электроноакцепторными группами.
- `AmidinoCenter`: `N-C(=N)-N` и близкие формы.
- `UreaLikeCenter`: мочевина, тиомочевина, гуанидин, карбаматные аналоги.
- `FormylationDonorCenter`: формамид, формиаты, активированные формильные переносчики.
- `CyclizableNucleophileElectrophilePair`: пара центров, которая может замкнуть кольцо допустимого размера.
- `ArylMigrationSite`: арильный или алкенильный фрагмент, способный участвовать в перегруппировке.

Где делать:

- распознавание: `molecule/functional_group.rs` и `molecule/reactive_site.rs`;
- типизированный доступ: `organic/centers.rs`;
- индексация для генерации: `organic/engine.rs`.

Критерии:

- центр не должен ссылаться на имя вещества;
- центр должен хранить индексы атомов, роль каждого атома и проверяемое окружение;
- все водороды только явные;
- невозможный центр не должен создаваться.

Пример формы:

```rust
pub(crate) struct BisNucleophileCenter<'a> {
    pub participant: SiteParticipant<'a>,
    pub first_nucleophile: usize,
    pub second_nucleophile: usize,
    pub bridge_atom: Option<usize>,
    pub class: BisNucleophileClass,
}

pub(crate) enum BisNucleophileClass {
    UreaLike,
    GuanidineLike,
    HydrazineLike,
    DiamineLike,
    AmidrazoneLike,
}
```

## Этап 2. Общий слой замыкания колец

Сейчас отдельные циклизации уже есть, но нужно выделить общий механизм. Он должен выполнять графовое преобразование и проверять результат.

Добавить общий строитель:

```rust
pub(crate) struct RingClosureSpec {
    pub prefix: &'static str,
    pub substrate_id: SubstanceId,
    pub nucleophile: usize,
    pub electrophile: usize,
    pub new_bond_order: f64,
    pub atoms_to_remove: Vec<usize>,
    pub bonds_to_change: Vec<BondOrderChange>,
    pub byproducts: Vec<SubstanceId>,
    pub allowed_ring_sizes: RangeInclusive<usize>,
    pub selectivity_profile: SelectivityProfile,
}
```

Он должен:

- проверить, что новая связь не создает невозможную валентность;
- проверить размер кольца;
- удалить только явно указанные уходящие атомы;
- вернуть ошибку при разрыве молекулы, если фрагмент не объявлен продуктом;
- после построения вызвать проверку графа и ароматичности;
- не знать слов `indole`, `purine`, `caffeine`, `uracil`.

Где делать:

- новый файл `organic/generators/ring_closure.rs`;
- существующие `generate_amidine_cyclization`, `generate_intramolecular_n_alkylation`, Paal-Knorr-замыкания постепенно перевести на этот строитель.

Критерии:

- старые тесты циклизаций проходят;
- новая общая функция покрыта тестами на пяти-, шести- и слитые кольца;
- невозможное кольцо возвращает ошибку или `Ok(None)` только если это химически допустимое отсутствие пути.

## Этап 3. C1/N-химия как источник азотных строительных блоков

Чтобы синтез сложных гетероциклов мог стартовать от простых веществ, нужны общие реакции малых C/N-фрагментов.

Добавить генераторы:

- гидролиз и аммонолиз изоцианатов, цианамидов, карбаматов;
- образование мочевинных и гуанидиновых фрагментов из аминов, цианамидов, изоцианатов, фосгена и аммиака;
- формамидирование аминов формильными переносчиками;
- образование гидразонов из карбонила и гидразинов;
- перенос формильной группы на амин или амидный азот, если центр позволяет.

Где делать:

- `organic/generators/c1_nitrogen.rs`;
- часть существующего `heteroatom.rs` можно оставить, но общие C1/N-реакции лучше вынести из него.

Критерии:

- из простых аминов можно получить уреа- и гуанидиноподобные центры;
- формамидный фрагмент не зашит графом, а строится переносом `C(=O)H`;
- все реакции сохраняют массу и заряд;
- условия сухости, кислоты, основания и воды описаны через существующие `ReactionCondition`.

## Этап 4. Обобщенные пиримидиновые и урациловые сборки

Нужно закрыть путь к пиримидиндионам и урацилоподобным ядрам без частного `uracil`-генератора.

Добавить генератор:

```rust
generate_bis_nucleophile_dicarbonyl_condensation(
    nucleophile: &BisNucleophileCenter,
    electrophile: &DicarbonylElectrophileCenter,
    resolver: &mut DerivedSubstanceResolver,
) -> ChemistryResult<Vec<Reaction>>
```

Он должен покрывать:

- мочевина/тиомочевина/гуанидин + 1,3-дикарбонил;
- образование шести-членного N-гетероцикла;
- потерю воды, спирта или аммиака только если уходящая группа реально есть;
- разные продукты по позициям, если дикарбонил несимметричен;
- таутомеризацию/ароматизацию через существующие правила графа, а не замену на готовый шаблон.

Где делать:

- `organic/generators/heterocycle.rs`;
- центры в `organic/centers.rs`;
- подключение в `organic/engine.rs`.

Критерии:

- из подходящего 1,3-дикарбонила и мочевинного центра получается пиримидиндион;
- из гуанидинового центра получается аминопиримидиновый аналог;
- если продукт имеет амидные N-H, они распознаются как `AmideNitrogen`;
- планировщик видит путь от простых предшественников до урацилоподобного ядра.

## Этап 5. Обобщенные пуриновые сборки

Часть уже есть: амидиновая циклизация может закрывать имидазольное кольцо на пиримидиндионе. Надо не зафиксировать один тест, а сделать слой полным.

Доработать:

- распознавание 4,5-диамино/амино-формамидо расположений на N-гетероциклах;
- формамидирование аминогруппы на гетероцикле;
- внутримолекулярную амидиновую циклизацию через общий `ring_closure`;
- сохранение карбонилов и амидных N-H;
- корректную ароматизацию слитого бицикла.

Где делать:

- `organic/generators/cyclization.rs`;
- `organic/generators/heterocycle.rs`;
- `molecule/aromatic_perception.rs`, если слитые N-гетероциклы распознаются неполно.

Критерии:

- `6-amino-5-formamidopyrimidinedione` закрывается в ксантиноподобный скелет;
- аналогичные аминопиримидины закрываются в другие пуриноподобные структуры;
- нет проверки имени вещества;
- тест проверяет не имя продукта, а графовые признаки: два карбонила, слитое кольцо, амидные азоты, ароматичность.

## Этап 6. N-алкилирование имидов, амидов и гетероароматических N-H

Часть уже есть через `generate_amide_n_alkylation`. Нужно расширить это до общего слоя.

Доработать:

- отделить `AmideNitrogen`, `ImideNitrogen`, `LactamNitrogen`, `AzaAromaticNitrogenH`;
- для каждого центра указать кислотность, нуклеофильность и требования к основанию;
- поддержать алкилгалогениды, сульфонаты и другие реальные уходящие группы, если они есть в модели;
- не использовать “N в ксантине” как особый случай.

Где делать:

- `molecule/functional_group.rs`;
- `organic/centers.rs`;
- `organic/generators/substitution.rs`.

Критерии:

- урацилоподобные, ксантиноподобные и обычные имиды алкилируются одним механизмом;
- метилирование ксантина в кофеин является частным случаем;
- если у азота нет явного H, реакция не создается;
- если среда не основная, реакция существует, но идет плохо через селективность, а не исчезает из пространства.

## Этап 7. Обобщенная индольная химия

Не добавлять “синтез индола”. Добавить механистические классы, которые могут дать индолы и замещенные индолы.

Первый слой: гидразонная арильная аннеляция.

Центры:

- `HydrazoneCenter`;
- `ArylHydrazoneCenter`;
- `ArylMigrationSite`;
- `CyclizableHydrazoneAnnulationSite`.

Генератор:

```rust
generate_hydrazone_aryl_annulation(
    hydrazone: &ArylHydrazoneCenter,
    resolver: &mut DerivedSubstanceResolver,
) -> ChemistryResult<Vec<Reaction>>
```

Он должен:

- работать с арил- и гетероарилгидразонами;
- проверять, что мигрирующий фрагмент действительно связан с гидразонным азотом;
- строить новую C-C или C-N связь графовой операцией;
- удалять аммиак/амин/воду только если эти атомы реально образуют уходящий фрагмент;
- создавать каналы для разных позиций, если окружение несимметрично;
- использовать кислотные условия и температурный профиль;
- возвращать индол, азаиндол, бензоиндол и замещенные варианты как продукты одного класса.

Второй слой:

- внутримолекулярное закрытие орто-аминоарил-карбонильных и орто-галогенарил-алкинильных систем;
- металлокаталитические варианты позже, только если есть общий слой металлокатализа C-C/C-N.

Где делать:

- `organic/generators/annulation.rs`;
- `organic/heterocycle.rs`;
- `molecule/aromatic_perception.rs`.

Критерии:

- фенилгидразон подходящего карбонила дает индольное ядро;
- замещенный арилгидразон дает замещенный индол без отдельного кода;
- гетероарилгидразон дает азаиндольный аналог, если валентность и ароматичность проходят;
- невозможная миграция дает понятную ошибку или отсутствие реакции.

## Этап 8. Общая классификация гетероциклических продуктов

Нужен не список молекул, а слой анализа результата.

Добавить:

- `HeterocycleDescriptor`;
- `RingNitrogenClass`;
- `FusedRingSystem`;
- `AromaticHeterocycleClass`;
- функции анализа колец после построения графа.

Назначение:

- понять, какие N являются пиридиновыми, пиррольными, амидными, имидными;
- отличить нуклеофильный N-H от слабого амидного N-H;
- передавать информацию в селективность и планировщик;
- не использовать имя вещества для решения, “можно ли дальше алкилировать/ацилировать/окислять”.

Где делать:

- новый `organic/heterocycle.rs`;
- подключение к `molecule/reactive_site.rs` и `organic/centers.rs`.

Критерии:

- пиррол, пиридин, имидазол, пиримидин, пуриноподобные слитые системы классифицируются по графу;
- классификация не меняет вещество, а только описывает реакционные центры;
- ошибки классификации не маскируются.

## Этап 9. Планировщик как проверка дыр

Планировщик не должен мутировать основной реестр и не должен быть кнопкой синтеза. Но он должен находить, где цепь обрывается.

Доработать `synthesis.rs`:

- добавить отчет о первом недостижимом классе центра;
- показывать, какие предшественники нужны для следующего шага;
- отличать “нет реакции в модели” от “нет исходных веществ”;
- уметь искать путь к динамической структуре через обобщенные генераторы;
- не расширять весь химический мир без входных веществ;
- использовать собственное рабочее состояние.

Пример диагностики:

```text
Цель: caffeine-like structure
Найден верхний путь:
  xanthine-like imide N-H + methyl halide -> methylated product
  amidine cyclization -> xanthine-like core
Обрыв:
  не найден путь к 6-amino-5-formamidopyrimidinedione
Нужные классы:
  pyrimidinedione precursor
  formylation donor
  aminated heterocycle
```

Критерии:

- для кофеиноподобной цели планировщик показывает верхние доступные шаги и обрыв ниже;
- после добавления пиримидиновых генераторов обрыв смещается к C1/N-строителям;
- после добавления C1/N-строителей появляется полный путь из простых веществ, если он реально поддержан.

## Этап 10. Полное закрытие кофеинового семейства

Это не отдельный генератор кофеина. Это проверочный сценарий, что общие слои работают вместе.

Нужный путь по классам:

1. простые C/N-строители дают мочевинные, гуанидиновые, формамидные и цианамидные центры;
2. C-C химия дает 1,3-дикарбонильный или эквивалентный электрофильный фрагмент;
3. би-нуклеофил + дикарбонил дает пиримидиндион;
4. формамидирование/аминирование дает предшественник для слитого имидазола;
5. амидиновая циклизация дает ксантиноподобный скелет;
6. обобщенное N-алкилирование дает кофеиноподобные метилированные продукты.

Тесты не должны проверять имя `caffeine`. Они должны проверять:

- молекулярную формулу целевого графа;
- наличие пуринового слитого скелета;
- три N-метильных заместителя;
- два карбонила;
- отсутствие лишних зарядов;
- сохранение массы и заряда на каждом шаге.

Если позже будет добавлено статическое вещество `caffeine`, динамический результат должен совпасть с ним через канонический граф.

## Этап 11. Полное закрытие индольного семейства

Проверочный сценарий для обобщенной аннеляции.

Пути по классам:

- карбонил + гидразин/арилгидразин -> гидразон;
- гидразонная арильная аннеляция -> индольное или азаиндольное ядро;
- ароматическое замещение и боковая функционализация дают замещенные индолы;
- N-алкилирование/ацилирование работает через общие N-центры.

Тесты:

- незамещенный индольный скелет;
- замещенный индол из замещенного арилгидразона;
- азаиндол из гетероарилгидразона, если граф проходит проверки;
- невозможный гидразон не дает реакции;
- продукты классифицируются как ароматические N-гетероциклы.

## Этап 12. Расширение до общей гетероциклической органики

После кофеина и индола не останавливаемся на них. Те же слои должны покрыть:

- пирролы;
- фураны;
- тиофены;
- имидазолы;
- оксазолы;
- тиазолы;
- пиридины;
- пиримидины;
- пиразины;
- хинолины;
- изохинолины;
- бензимидазолы;
- пуриноподобные системы.

Добавлять новые семейства можно только через:

- новый реакционный центр;
- новый общий механизм аннеляции или конденсации;
- расширение классификатора гетероциклов;
- тесты на несколько представителей класса.

Запрещено добавлять “один генератор под одну молекулу”.

## Правила реализации для исполнителя

1. Не проверять `SubstanceId`, если это не внешний реагент каталога вроде воды, протона, аммиака, кислорода или известного простого реагента.
2. Не вставлять готовый граф продукта. Продукт строится только операциями `MolecularEditor`.
3. Не использовать молярную массу или строковую формулу как источник структуры.
4. Не использовать скрытые водороды.
5. Не возвращать `Ok(None)` при внутренней ошибке графа. Ошибка должна быть явной.
6. Не отключать проверку массы и заряда.
7. Не добавлять маленькие пределы итераций как способ завершения. Завершение должно идти через ключи вещества, генератора и центра.
8. Не смешивать распознавание центра и преобразование продукта в одной большой функции.
9. Каждый новый генератор должен иметь тест минимум на два разных субстрата одного класса.
10. Каждый новый центр должен иметь отрицательный тест на похожий, но неподходящий фрагмент.
11. Если реакция зависит от среды, генератор создает структурно возможную реакцию, а скорость и распределение регулируются селективностью.
12. Если возникает соблазн написать имя целевого вещества в коде, нужно остановиться и выделить более общий центр.

## Пример структуры генератора

```rust
pub(crate) fn generate_bis_nucleophile_dicarbonyl_condensation(
    nucleophile: &BisNucleophileCenter<'_>,
    electrophile: &DicarbonylElectrophileCenter<'_>,
    resolver: &mut DerivedSubstanceResolver,
) -> ChemistryResult<Vec<Reaction>> {
    if !nucleophile.can_close_with(electrophile) {
        return Ok(Vec::new());
    }

    let mut products = Vec::new();
    for topology in possible_condensation_topologies(nucleophile, electrophile)? {
        let product_structure = build_condensed_ring(nucleophile, electrophile, &topology)?;
        let product_id = resolver.resolve(product_structure)?;
        products.push(build_condensation_reaction(
            nucleophile,
            electrophile,
            product_id,
            topology,
        )?);
    }

    Ok(products)
}
```

В этой форме нет имени вещества и нет готового продукта. Есть только центры, допустимые топологии и проверяемое преобразование графа.

## Порядок работ

1. Ввести новые центры C/N-строителей и тесты распознавания.
2. Вынести общий строитель замыкания колец.
3. Перевести существующие циклизации на общий строитель без изменения поведения.
4. Добавить C1/N-химию малых строительных блоков.
5. Добавить би-нуклеофил + дикарбонил конденсации.
6. Доработать пуриновые циклизации и N-алкилирование.
7. Добавить гидразонную арильную аннеляцию.
8. Добавить классификатор гетероциклических продуктов.
9. Доработать планировщик, чтобы он показывал обрывы пути.
10. Добавить проверочный путь к кофеиноподобной структуре.
11. Добавить проверочный путь к индольному семейству.
12. Расширить тот же подход на остальные N/O/S-гетероциклы.

## Проверка готовности

Минимальная готовность:

- нет частных генераторов под кофеин, индол, урацил или ксантин;
- есть типизированные центры C/N-строителей;
- есть общие конденсации и аннеляции;
- планировщик показывает обрыв, если полного пути еще нет.

Хорошая готовность:

- кофеиноподобная структура строится через общие реакции;
- индольное семейство строится через общие реакции;
- планировщик объясняет путь и условия;
- все новые продукты проходят канонизацию, ароматичность, массу, заряд, валентность и стереохимию.

Полная готовность:

- основные малые N/O/S-гетероциклы доступны из общих центров;
- новые замещенные аналоги не требуют нового кода;
- добавление нового исходного вещества расширяет пространство синтеза автоматически;
- тесты покрывают не только один целевой продукт, а классы соединений.
